## 1. 布局与信息架构


| 区域  | 要求                                          |
| --- | ------------------------------------------- |
| 左侧  | 任务列表：任务名称可编辑；支持行排序（如左侧拖动手柄）。                |
| 右侧  | **可横向滚动**的时间线：顶部为日期刻度，下方每行一条任务轨道与色块。        |
| 工具区 | 提供「显示星期」开关、「回到今天」、添加任务；以及 CSV / Excel 导入导出。 |


---

## 2. 时间线：无限延伸与锚定「今天」

1. **横向无限滚动**：用户向左/右滚动时间轴时，在接近边缘时**自动向前/后追加日期**，使时间轴在体验上可连续延伸，不因固定起止日而卡住。
2. **默认视口锚定今天**：首次进入或合理重置时，视口应落在「今天」附近（例如今天左右各一段范围），并可通过「回到今天」一键滚回。
3. **与任务拖拽联动**：拖拽任务条整体平移、或拖动左右手柄改起止日时，若指针贴近时间轴可视区左右边缘，应能**自动滚动时间轴**并在需要时**继续扩展日期范围**，保证任务日期可调到当前视口之外。

---

## 3. 任务条交互（保留原有能力）

1. **整体平移**：拖动任务条主体，保持跨度不变，改开始/结束日。
2. **改跨度**：拖动左/右边缘手柄，分别调整开始日或结束日。
3. **行排序**：通过左侧手柄拖拽调整任务行顺序（与时间线独立）。
4. **颜色**：每条任务保留**可区分的颜色**（如随机或分配色相），用于多条任务并行时的识别。

---

## 4. 日期展示规范

1. **时间轴刻度**：日期文案统一为 `**月/日`**，**不补零**，例如：`3/24`、`12/1`、`1/26`。
2. **星期**：可选展示「周×」，由「显示星期」开关控制；与刻度对齐、易扫读。

---

## 5. 任务条上的文案与 Tooltip

1. **条内短文案**：在条上展示**起止日期 + 任务名**（或稿中规定的组合），字号与截断方式遵循 Figma。
2. **过长内容**：条内展示不下时，**完整信息通过 Tooltip（如原生 `title` 或设计稿指定组件）** 展示，至少包含：**任务名 + 起止日期**（格式与条内日期规则一致）。

---

## 6. 数据：导入、导出与本地持久化

1. **导出**：支持 **CSV** 与 **Excel（.xlsx）**。
2. **导入**：支持从 CSV / Excel 导入并映射为任务（列规则与现有实现或单独「导入说明」一致）。
3. **持久化**：任务与时间线相关状态保存在本机浏览器（具体 key 以实现为准），刷新不丢。

---

## 验收检查清单（对照 Figma + 上文）

- 时间轴可左右无限滚动态延伸，滚动与拖拽边缘行为顺畅、无乱跳。
- 默认进入或「回到今天」时，今天落在合理可视位置。
- 任务条可平移、左右缩放改日期；行可排序；颜色多样。
- 刻度为 `月/日` 且不补零；星期可选显示。
- 长任务名/长文案有条内截断 + Tooltip 完整信息。
- CSV / Excel 导入导出可用；数据本地可恢复。
